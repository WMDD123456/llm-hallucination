# 基于事实性问答场景的大模型幻觉检测、评测与缓解方法研究

## 一、研究背景

大语言模型存在"幻觉"问题——生成的内容语法通顺但与事实不符。在医疗、法律、教育等对事实准确性要求高的场景中，幻觉严重限制了 LLM 的实际应用。

本课题针对中文事实性问答场景，设计四组消融实验（G1–G4），逐步叠加缓解策略，系统评估不同方案的效果。

## 二、方案设计

### 整体架构

```
评测集 (150题) ──→ G1 裸模 ──→ G2 +基础RAG ──→ G3 +异构集成 ──→ G4 全链路
                     │              │                 │                │
                     ▼              ▼                 ▼                ▼
                LLM-as-Judge 统一评测 (DeepSeek-Chat 裁判)
                     │
                     ▼
              HR / FA / RA / AA / MG 五指标 + 幻觉五分类
```

### 四组消融实验

| 实验组 | 策略 | 模型 | 说明 |
|--------|------|------|------|
| G1 裸模型基线 | 直接问答 | DeepSeek-Chat ×1 | 不引入外部证据和缓解机制 |
| G2 +基础 RAG | 检索增强 | DeepSeek-Chat ×1 | 检索百度百科 evidence 注入 prompt，证据不足时允许拒答 |
| G3 +异构多模型投票 | 多模型投票 | DS + GLM + Qwen-Max + Qwen-Turbo | 四模型在相同检索文本条件下并行生成候选答案，直接投票选最终答案 |
| G4 全链路缓解方案 | 全链路 | 同上 4 模型 | 路由分类 → CoT 推理 → 结构化 JSON 输出 → 四模型并行生成 → Jaccard 相似度加权投票 → 不确定性拒答 |

**G1（裸模型基线）**：使用 DeepSeek-Chat 直接回答，不引入外部证据和缓解机制，作为无基线、无干预条件下的幻觉率参照。

**G2（+基础 RAG）**：使用 DeepSeek-Chat 基于基础 RAG 检索文本回答，证据不足时允许拒答。验证单模型条件下 RAG 检索增强对幻觉的抑制效果。

**G3（+异构多模型投票）**：使用 DeepSeek-Chat、GLM-4-Flash、Qwen-Max、Qwen-Turbo 四个模型在相同检索文本条件下并行生成候选答案，直接投票选择最终答案。验证多模型候选答案之间的一致性判断是否能进一步降低幻觉。

**G4（全链路缓解方案）**：在 G3 基础上增加问题路由、CoT 推理、结构化 JSON 输出、Jaccard 相似度加权投票和不确定性拒答机制。用户提问 → Qwen-Turbo 路由分类（简单问题/复杂问题/知识库外问题）→ 基础 RAG 检索相关文本 → 四模型并行生成（prompt 要求 CoT 分析 + 结构化 JSON）→ 提取最终答案 → 标准化处理 → 计算 Jaccard 相似度矩阵 → 相似度加权投票 → 若投票一致性低于阈值或多数模型判断资料不足则触发不确定性拒答 → 事实一致性比对 → 标注与统计。

**对比逻辑**：
- G1 vs G2：RAG 单独有效吗？
- G2 vs G3：多模型投票比单模型 RAG 更好吗？
- G3 vs G4：加路由 + CoT + 拒答有额外收益吗？

## 三、数据集与知识库

从公开百科、权威网页和修订后的评测集证据文本中整理知识材料，经文本清洗、切分和格式统一后构建 FAISS 向量索引。

评测集共包含 150 道中文事实性问答题，来源于 Chinese-SimpleQA 和 WebQA 两个公开数据集，覆盖人物、时间、地点、数值、事件、专业知识六大领域。实验前对部分存在答案错误、证据冲突或证据缺失的条目进行了人工修订，以提高评测结果的可信度。知识库主要用于支持 G2、G3、G4 的 RAG 检索增强实验，并保证不同实验组在相同数据条件下进行公平比较。

基础 RAG 使用 LangChain 框架 + FAISS 向量检索，嵌入模型选用 BAAI/bge-large-zh-v1.5。知识库文本经清洗与切分后构建向量索引，用户问题输入后先进行相似度检索，再将检索结果与问题共同输入模型生成答案。若检索文本不足以支持明确回答，则允许模型拒答。本项目以基础 RAG 为主要检索底座，G4 的改进重点放在问题路由、CoT 推理、结构化输出、多模型投票和不确定性拒答上。

## 四、评测方法

### 评测指标

| 指标 | 含义 | 公式 | 方向 |
|------|------|------|------|
| HR (Hallucination Rate) | 幻觉回答占比 | 幻觉数 / 总数 × 100% | ↓ 越低越好 |
| FA (Factual Accuracy) | 事实正确占比 | 正确数 / 总数 × 100% | ↑ 越高越好 |
| RA (Refusal Rate) | 拒答占比 | 拒答数 / 总数 × 100% | — 适当即可 |
| AA (Answered Accuracy) | 敢答准确率 | FA / (FA + HR) × 100% | ↑ 越高越好 |
| MG (Mitigation Gain) | 缓解增益 | (G1_HR − Gx_HR) / G1_HR × 100% | ↑ 越高越好 |

### LLM-as-Judge

以 LLM-as-judge 自动评测为主，结合关键词规则校验和人工抽查作为回退机制。裁判模型采用 DeepSeek-Chat，并使用宽松版评测 prompt，对同义词、简称、书名号差异、年份后缀、中英文别名和格式差异进行容忍，避免将语义等价答案误判为错误。

### 幻觉类型识别

在判定为幻觉后，进一步细分为五类：

| 类型 | 定义 | 示例 |
|------|------|------|
| 事实错误 | 实体信息与标准答案不符 | 人名、地名、作品名等实体错误 |
| 关系错误 | 实体间关系描述错误 | 归属、因果、从属关系错误 |
| 数值/时间错误 | 数字、日期、年份等不一致 | "1920年" 误为 "1921年" |
| 信息编造 | 内容无任何依据，凭空生成 | 编造不存在的文献名、数据 |
| 冲突/逻辑矛盾 | 回答内自相矛盾或与常识冲突 | 前后说法不一致 |

## 五、实验结果（2026-06-08）

### 全量 150 题

| 组别 | HR ↓ | FA ↑ | RA | AA ↑ |
|------|------|------|-----|------|
| **G1** 裸模 | 20.0% | 77.3% | 2.7% | 79% |
| **G2** +RAG | **0.7%** | **83.3%** | 16.0% | **99%** |
| **G3** +异构集成 | 1.3% | 81.3% | 17.3% | 98% |
| **G4** 全链路 | 4.0% | 79.3% | 16.7% | 95% |

### 复杂推理题子集 (10 题)

| 组别 | FA ↑ | RA |
|------|------|-----|
| G2 | 50.0% | 50.0% |
| **G4** | **70.0%** | 30.0% |

### 关键发现

1. **RAG 是最大贡献因子**：G1→G2 幻觉率从 20% 降至 0.7%（降幅 96.5%），"不知道就拒答"远好于"瞎编"
2. **多模型投票不是万能药**：数据集 93% 为简单事实题，G3/G4 的投票反而内耗；但在 10 道复杂推理题上 G4 显著优于 G2（FA +20pp）
3. **策略应与题型匹配**：简单题用 G2（RAG 足够），复杂推理题用 G4（多视角验证）

## 六、技术栈

| 层次 | 选型 |
|------|------|
| LLM 调用 | DeepSeek-Chat, GLM-4-Flash, Qwen-Max, Qwen-Turbo |
| RAG 检索 | FAISS + BM25 Hybrid + bge-large-zh-v1.5 + 百度百科 300 篇词条 |
| 自动评测 | LLM-as-Judge (DeepSeek-Chat) + 幻觉五分类 |
| 集成投票 | Jaccard 相似度加权投票 + 不确定性阈值拒答 |
| 可视化 | matplotlib |

## 八、运行

```bash
pip install -r requirements.txt
cp .env.example .env   # 编辑填入 API key

# 全部实验
python run.py --groups G1 G2 G3 G4

# 仅评测已有结果
python eval_llm_judge.py data/results/results_xxx.json

# 画图
python visualize_results.py
```
