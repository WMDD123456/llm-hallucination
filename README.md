# 大模型幻觉检测与缓解方案研究

## 课题背景

大模型有时候会胡说八道，答案看着挺像回事，实际上事实是错的，这就是幻觉问题。本课题想搞清楚两件事：什么时候模型会幻觉？怎么减少幻觉？

## 做了什么

做了一个实验框架，用 20 道常识题去测模型。测试分成 5 种方法：

G1 直接问：不给任何参考，看模型自己能答对多少。

G2 给参考材料：用 RAG 从维基百科检索相关内容塞进提示里。

G3 要求标注来源：在 G2 基础上，让模型回答时标明引用了哪条。

G4 让模型自查：在 G2 基础上，让模型答完自己检查一遍。

G5 全套：参考材料加标注来源加自查，三样都上。

最后用 LLM 当裁判来判断每一条回答是不是幻觉，而不是简单地去匹配关键词。

## 怎么跑

安装依赖：pip install -r requirements.txt

配好 API key 后：python run.py

也可以用 --models 指定模型、--groups 指定实验组。

## 评测结果

目前用 DeepSeek-Chat 跑了 G1 基线。基线幻觉率比较高，不过这是好事，说明确实有幻觉需要解决，后面 G2 到 G5 的缓解方案才能看出效果。

## 碰到的问题

幻觉怎么判定是最大的坑。最开始用的关键词匹配，中文换个说法或者数字写法不一样就认不出来了，后来换成了 LLM 裁判才靠谱。

RAG 检索的质量也是个问题，知识库里没有的内容检索不到，模型就只能瞎猜。
